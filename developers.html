Getting started

Authentication

Every request is authenticated with a secret API key sent as a bearer token. You'll find your keys in your business account under Settings › Developer — a sk_test_ key for the sandbox and a sk_live_ key once you're ready. Keep the secret key server-side; never ship it in browser code.

Authorization: Bearer sk_live_3f9c2a7b41e84d06b8ce55a1
One account, one network. The API books into the same scheduled collection as the dashboard — orders you push land in the day's batch and go out on your fixed pickup, delivered by 6pm.
Getting started

How it fits together

Three calls cover the whole journey — from a shopper choosing same-day at your checkout to a delivered photo on the doorstep.

  • At checkout — call POST /rates with the delivery Eircode and the cart's size. If Molli is eligible you get a live price to show as a shipping option; if it's out of zone, too big, or past the cut-off, the rate simply isn't returned.
  • On payment — call POST /shipments to drop the order into today's batch. You get back a tracking number and the collection it's assigned to.
  • Then — we send webhook events as the parcel is collected, sorted, out for delivery, and delivered, so you can keep your customer updated under your own brand.
Endpoints

Get a rate

Returns the same-day services available for a specific address and parcel, priced live by zone. Call this at checkout to decide whether to show Molli as a shipping option.

POST/v1/rates
curl https://api.molli.ie/v1/rates \
  -H "Authorization: Bearer sk_live_3f9c2a7b41e8" \
  -H "Content-Type: application/json" \
  -d '{
    "delivery":   { "eircode": "D06 X235" },
    "collection": { "eircode": "D08 CTX3" },
    "parcel":     { "weight_g": 480, "length_cm": 22, "width_cm": 14, "height_cm": 6 }
  }'
const res = await fetch("https://api.molli.ie/v1/rates", {
  method: "POST",
  headers: {
    "Authorization": `Bearer ${process.env.MOLLI_KEY}`,
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    delivery:   { eircode: "D06 X235" },
    collection: { eircode: "D08 CTX3" },
    parcel:     { weight_g: 480, length_cm: 22, width_cm: 14, height_cm: 6 }
  })
});
const { rates } = await res.json();
{
  "rates": [
    {
      "id":          "rate_same_day",
      "service":     "same_day",
      "name":        "Same-day by Molli",
      "price_eur":   7.99,
      "zone":        "city_centre",
      "order_before": "14:00",
      "delivered_by": "18:00"
    }
  ],
  "eligible": true   // false + "reason" when out of zone / oversize / past cut-off
}
Eligibility is handled for you — zone by Eircode, size against what a bike or van can carry, and the live cut-off. No same-day rate comes back when it can't be delivered today, so you never show a promise you can't keep.

Create a shipment

Books the order into today's scheduled collection and returns a tracking number and label. Send it once the customer has paid.

POST/v1/shipments
rate_idREQUIREDstring
The id from the rate you showed at checkout.
deliveryREQUIREDobject
Recipient name, phone, address, eircode.
parcelREQUIREDobject
Weight and dimensions, as sent to /rates.
referencestring
Your own order number — echoed back on every webhook.
optionsobject
Per-parcel extras. insurance_eur declares a value to cover (up to €500). Photo proof of delivery and an en-route SMS to the recipient are included on every shipment — no flag needed.
Insure per order, not per account. Pass insurance_eur only on the parcels that need cover (€4.99, up to €500) — everything else ships with standard included cover.
curl https://api.molli.ie/v1/shipments \
  -H "Authorization: Bearer sk_live_3f9c2a7b41e8" \
  -H "Content-Type: application/json" \
  -d '{
    "rate_id": "rate_same_day",
    "reference": "GRA-10482",
    "delivery": {
      "name": "Niamh O'\''Sullivan",
      "phone": "+353 87 555 0142",
      "address": "14 Grove Park, Rathmines",
      "eircode": "D06 X235"
    },
    "parcel": { "weight_g": 480, "length_cm": 22, "width_cm": 14, "height_cm": 6 },
    "options": { "insurance_eur": 250 }   // omit to ship with standard included cover
  }'
{
  "id":         "shp_8Kd2mQ9vX1",
  "tracking":   "MOLLI-94821-D",
  "status":     "booked",
  "reference":  "GRA-10482",
  "collection": {
    "date":   "2026-07-07",
    "window": "13:00–13:30"   // your scheduled slot
  },
  "delivered_by": "18:00",
  "label_url":    "https://api.molli.ie/labels/shp_8Kd2mQ9vX1.pdf",
  "track_url":    "https://molli.ie/track/MOLLI-94821-D"
}

Track a shipment

Fetch the current status and timeline for a shipment. Most integrations rely on webhooks instead of polling, but this is handy for an order page or a support lookup.

GET/v1/shipments/{id}
curl https://api.molli.ie/v1/shipments/shp_8Kd2mQ9vX1 \
  -H "Authorization: Bearer sk_live_3f9c2a7b41e8"
{
  "id":       "shp_8Kd2mQ9vX1",
  "tracking": "MOLLI-94821-D",
  "status":   "out_for_delivery",
  "rider":    "Eamon",
  "eta":      "2026-07-07T16:42:00Z",
  "events": [
    { "status": "collected",        "at": "13:12" },
    { "status": "at_hub",           "at": "14:05" },
    { "status": "out_for_delivery", "at": "15:30" }
  ]
}
Events

Webhooks

Point a URL at Molli under Settings › Developer › Webhooks and we'll POST a signed event every time a shipment changes state. Each payload carries your reference so you can match it to your own order.

shipment.collectedPicked up on your scheduled run
shipment.at_hubSorted into a geographic run
shipment.out_for_deliveryWith a rider, on the local loop
shipment.deliveredDelivered — photo & signature attached
shipment.exceptionNobody home, refused, or address issue
{
  "event":     "shipment.delivered",
  "created":   "2026-07-07T16:48:11Z",
  "data": {
    "id":        "shp_8Kd2mQ9vX1",
    "tracking":  "MOLLI-94821-D",
    "reference": "GRA-10482",
    "proof": {
      "photo_url": "https://api.molli.ie/proof/shp_8Kd2mQ9vX1.jpg",
      "signed_by": "N. O'Sullivan"
    }
  }
}
Reference

Test mode

Use your sk_test_ key to build against the full API without booking a real collection or being charged. Test shipments move through every webhook state on an accelerated timeline so you can watch the whole journey in minutes.

  • Eircode D02 AF30 always resolves to the city-centre zone; D24 KV82 to greater Dublin.
  • A parcel over the carry limits returns an ineligible rate with reason: "oversize".
  • Set the delivery name to Test Exception to trigger a shipment.exception event.
